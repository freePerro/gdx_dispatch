DispatchApp Public API

The GDX Public API lets you integrate your workflows with external systems, build custom dashboards, or automate job and customer management from your own code.

Base URL: https://api.example.com

Authentication

All public API endpoints require an API key. Pass it in the X-API-Key HTTP header with every request.

X-API-Key: gdx_live_<your-key>
Key created once. The full API key is only shown at creation time. Copy and store it securely — you cannot retrieve it again. If lost, revoke the key and create a new one.

Scopes

Each API key is granted one or more scopes that control what it can access:

ScopeAccess granted
read:jobsList and fetch jobs
write:jobsCreate and update jobs
read:customersList and fetch customers
write:customersCreate customers

Rate Limits

API keys are rate-limited to 60 requests per minute. When the limit is exceeded the API returns HTTP 429 Too Many Requests. Retry after 60 seconds.

Rate limits are per API key, not per IP address. Each key has its own 60 req/min window.

Jobs

MethodPathScopeDescription
GET /v1/jobs read:jobs List all active jobs (up to 200)
POST /v1/jobs write:jobs Create a new job
GET /v1/jobs/{id} read:jobs Get a single job by ID
PATCH /v1/jobs/{id} write:jobs Update job status, title, or scheduled time

Response format

All responses are wrapped in a standard envelope:

{
  "data": { /* resource or array */ },
  "meta": {
    "request_id": "7e4a...",
    "timestamp": "2026-03-26T15:00:00.123456Z"
  }
}

Errors follow the FastAPI default: {"detail": "message"}.

cURL examples

List jobs

curl -s https://api.example.com/v1/jobs \
  -H "X-API-Key: gdx_live_your_key_here"

Create a job

curl -s -X POST https://api.example.com/v1/jobs \
  -H "X-API-Key: gdx_live_your_key_here" \
  -H "Content-Type: application/json" \
  -d '{"title": "Spring tune-up", "status": "Scheduled"}'

Create a customer

curl -s -X POST https://api.example.com/v1/customers \
  -H "X-API-Key: gdx_live_your_key_here" \
  -H "Content-Type: application/json" \
  -d '{"name": "Jane Smith", "email": "jane@example.com", "phone": "555-0100"}'

Update job status

curl -s -X PATCH https://api.example.com/v1/jobs/<job-id> \
  -H "X-API-Key: gdx_live_your_key_here" \
  -H "Content-Type: application/json" \
  -d '{"status": "Completed"}'

Client class

import requests

class TGDClient:
    def __init__(self, api_key: str, base_url: str = "https://api.example.com"):
        self.base = base_url
        self.headers = {"X-API-Key": api_key, "Content-Type": "application/json"}

    def list_jobs(self):
        r = requests.get(f"{self.base}/v1/jobs", headers=self.headers)
        r.raise_for_status()
        return r.json()["data"]

    def create_job(self, title: str, status: str = "Scheduled", customer_id: str = None):
        payload = {"title": title, "status": status}
        if customer_id:
            payload["customer_id"] = customer_id
        r = requests.post(f"{self.base}/v1/jobs", json=payload, headers=self.headers)
        r.raise_for_status()
        return r.json()["data"]

    def list_customers(self):
        r = requests.get(f"{self.base}/v1/customers", headers=self.headers)
        r.raise_for_status()
        return r.json()["data"]

    def create_customer(self, name: str, email: str = None, phone: str = None):
        payload = {"name": name}
        if email: payload["email"] = email
        if phone: payload["phone"] = phone
        r = requests.post(f"{self.base}/v1/customers", json=payload, headers=self.headers)
        r.raise_for_status()
        return r.json()["data"]


# Usage
client = TGDClient("gdx_live_your_key_here")
jobs = client.list_jobs()
print(jobs)

new_job = client.create_job("Spring tune-up", status="Scheduled")
print(new_job["id"])
